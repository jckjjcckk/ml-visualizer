1. Product vision

You’re building:

An interactive machine learning visualization toolbox

Key characteristics:

* Playground-first, not lecture-first
* Structured by course (weeks → topics → tools)
* Designed for real users, not just personal notes
* Premium, product-quality UI from day one

⸻

2. Core UX philosophy

What users do

Open tool → understand quickly → interact → observe behavior

NOT:

Read long theory → then maybe interact

Content priority (V1)

Visualization + controls + guided intro

Deferred:

Heavy theory
Formulas
Lecture-style content

⸻

3. Information architecture

Hierarchy:

Week → Topic → Tool (visualizer)

Navigation:

* Linear sidebar (hideable)
* Global search across all tools
* Feels like:
    * a toolbox
    * but ordered like a course

⸻

4. Page structure (standardized)

Each tool page:

Top bar
Sidebar
Main visualization
Control panel
Guided intro overlay (first visit)

Layout concept:

Sidebar | Visualization
        | Controls

⸻

5. Interaction design

Depth:

Medium → Heavy

Features:

* Sliders (e.g. k in kNN)
* Step / play controls
* Reset state
* Preset datasets
* Later:
    * custom data upload
    * handwriting demo
    * comparisons

⸻

6. State philosophy

For now:

All state is local
Refresh resets everything

Later (with accounts):

Synced:

* progress
* quiz results
* bookmarks

NOT synced:

* visualization parameters
* temporary data
* tool state

⸻

7. Design direction

Target:

Dark-first
Minimal but not empty
Technical but premium
Apple-like polish

Key principles:

* Consistent spacing system
* High-quality typography
* Subtle animations (not flashy)
* Meaningful color usage
* Fast and smooth performance

Avoid:

* generic dashboards
* overly flashy styles
* “unfinished minimal”

⸻
